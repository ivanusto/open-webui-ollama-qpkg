ℹ️ 為什麼會看到這個畫面？

    

      本套件安裝後不會內含任何 AI 模型或應用程式本體，而是在背景請 Container Station
      下載官方的 ollama/ollama 與 ghcr.io/open-webui/open-webui 映像檔。
      在下載完成前，這個輕量狀態頁會暫時佔用網頁埠，讓你可以確認下載進度；
      下載完成後，套件會自動建立並啟動兩個容器，Open WebUI 的真正介面就會出現在同一個網址上，不需要手動重新整理設定。
    

  


  

    
⚡ GPU 加速

    

      套件會自動偵測 Container Station 是否已啟用 NVIDIA GPU runtime（需先在 App Center 安裝官方 NVIDIA GPU Driver，
      且機型支援獨立顯示卡）。偵測到才會將 --gpus all 傳給 Ollama 容器；沒有偵測到則自動以 CPU 模式運作，
      並在系統紀錄留下提示，套件本身在兩種情況下都能正常啟動。

💾 資料存放位置

    

      Ollama 下載的模型檔案通常單一檔案就有數 GB 到數十 GB，請確保
      模型儲存路徑 所在磁碟區有足夠空間。Open WebUI 的聊天記錄、上傳文件與 RAG
      向量資料庫則存放在 WebUI 資料路徑。兩者預設都在套件安裝目錄下的
      data/ 資料夾，重灌套件不會遺失資料。
    

    

      變更路徑：編輯 QPKG 安裝目錄下的 openwebui-ollama.conf，設定
      OLLAMA_DATA_PATH / WEBUI_DATA_PATH 後重新啟動本套件即可套用。
    

  


  

    
🚀 首次使用

    
    
      
  1. 安裝完成後，套件會於背景下載官方映像檔（視網速需數分鐘到數十分鐘，模型執行環境本體較大）。
    2. 
      
  2. 下載完成後開啟本頁（同一網址）即可看到 Open WebUI 登入畫面；第一個註冊的帳號會自動成為管理員。
    3. 
      
  3. Open WebUI 已預先設定好連到內部的 Ollama 服務（OLLAMA_BASE_URL 已自動指向同一私有網路內的 Ollama 容器），
        登入後直接在介面右上角「選擇模型」搜尋並下載模型即可開始對話。
    4. 
      
  4. 若要開放註冊給團隊其他成員，或改成邀請制，可在 Open WebUI 的「管理員設定」內調整，不需要修改本套件設定。
    5. 
    

    
下載進度：<QPKG 安裝目錄>/logs/pull.log；服務記錄：logs/openwebui-ollama.log。

  


  

    
🔧 常用維護指令（SSH）

    
/etc/init.d/openwebui-ollama.sh status    # 檢視狀態
/etc/init.d/openwebui-ollama.sh restart   # 重新啟動
/etc/init.d/openwebui-ollama.sh update    # 更新官方映像並重建容器（資料保留）
/etc/init.d/openwebui-ollama.sh diag      # 診斷 GPU / 映像檔 / 網路連線

  


  

    本套件僅包含管理指令碼與此狀態頁，不內含任何 Ollama 或 Open WebUI 程式本體；分別由
    Ollama 官方映像 與
    Open WebUI 官方映像 提供。
    套件結構參考 qnap-dev/containerized-qpkg。
