ℹ️ Why Am I Seeing This Page?

    

      This package does not ship any AI models or application binaries. Instead, it asks Container Station in the background
      to download the official ollama/ollama and ghcr.io/open-webui/open-webui images.
      Until the download finishes, this lightweight status page temporarily occupies the web port so you can watch the progress;
      once complete, the package automatically creates and starts both containers, and the real Open WebUI interface appears at this same URL — no manual reconfiguration needed.
    

  


  

    
⚡ GPU Acceleration

    

      The package automatically detects whether Container Station has the NVIDIA GPU runtime enabled (requires the official
      NVIDIA GPU Driver from App Center and a model that supports discrete GPUs). Only when detected does it pass
      --gpus all to the Ollama container; otherwise it automatically runs in CPU mode and leaves a note in the
      system log. The package starts up fine either way.

💾 Data Storage Locations

    

      Model files downloaded by Ollama are typically several GB to tens of GB each, so make sure the volume holding the
      Model Storage Path has enough free space. Open WebUI's chat history, uploaded documents, and RAG vector
      database are stored at the WebUI Data Path. Both default to the data/ folder inside the QPKG
      installation directory, so reinstalling the package does not lose data.
    

    

      To change paths: Edit openwebui-ollama.conf in the QPKG installation directory, set
      OLLAMA_DATA_PATH / WEBUI_DATA_PATH, then restart this package to apply.
    

  


  

    
🚀 Getting Started

    
    
      
  1. After installation, the package downloads the official images in the background (a few minutes to tens of minutes depending on network speed — the model runtime image is large).
    2. 
      
  2. Once the download completes, open this page (same URL) to see the Open WebUI login screen; the first account registered automatically becomes the administrator.
    3. 
      
  3. Open WebUI is pre-configured to reach the internal Ollama service (OLLAMA_BASE_URL already points to the Ollama container on the same private network). After logging in, use "Select a model" at the top of the interface to search for and download a model, then start chatting.
    4. 
      
  4. To open registration to other team members, or switch to invite-only, adjust it in Open WebUI's "Admin Settings" — no changes to this package are needed.
    5. 
    

    
Download progress: <QPKG Root>/logs/pull.log; Service log: logs/openwebui-ollama.log.

  


  

    
🔧 Useful Maintenance Commands (SSH)

    
/etc/init.d/openwebui-ollama.sh status    # Check status
/etc/init.d/openwebui-ollama.sh restart   # Restart
/etc/init.d/openwebui-ollama.sh update    # Update official images & rebuild containers (data is kept)
/etc/init.d/openwebui-ollama.sh diag      # Diagnose GPU / images / network connectivity
